## Findings and follow-ups

- **Subtree shares journal the whole filesystem.** A `FAN_MARK_FILESYSTEM` mark
  sees every fs event; the engine now filters to the share subtree by path
  containment (open_by_handle + /proc/self/fd readlink, cached). Whole-mount
  shares (the recommended layout) skip the filter entirely. The filter cost is
  one open+readlink per distinct out-of-subtree directory, then cached — fine for
  steady state, but a dedicated ext4 volume per share-set is still preferable.
- **macOS NFS mount worked without an explicit sudo prompt in this run** (hostd
  invoked `mount_nfs` as the user against a user-owned mountpoint on loopback).
  This was environment-dependent; the release includes a documented privileged-helper
  path so `mist mount` is reliable across machines rather than relying on it.
- **Read-only mount + `mdutil -i off`** neutralize Apple-metadata pollution for
  the read-only surface; the writable side-store (absorbing `.DS_Store`/AppleDouble) ships in
  the release.
- **Journal lag was well under the 1.5 s settle used in the script** — a tighter
  latency measurement (guest-touch → host-observes) is worth adding as a metric.
- **NFSv4.1 + delegations** remain the path to driving cold-scan revalidation
  RPCs to zero on the hot set; Mist's v3 already answers all metadata from the replica.
