## Verdict — write-path hardening: all four deferred items delivered

1. **Side-store (design 06) PROVEN on hardware.** `xattr -w` on a mount file round-trips
   (`xattr -p` returns the value) while `mist ls` shows the guest tree clean — no `._*`, no
   `.DS_Store` ever reach the guest; listings on the Mac side hide them too (direct lookup
   serves them). Persistent across hostd restarts (postcard store per share);
   `.metadata_never_index` synthesized at the root.
2. **Conflict detection PROVEN** (`conflict detection OK, total=11`): a 1 Hz guest writer +
   a Mac write to the same file produce visible `mist conflicts` rows in both directions
   (mac-over-guest / guest-over-mac), last-close-wins applied. **The churn test caught a real
   journal baseline bug**: with no journal batch buffered during a fast seed, hostd baselined the expected
   seq at 1 while the guest's batcher was already at N → every batch "gapped" → permanent
   resync loop (live journal never applied). Fixed with SEQ_ADOPT: adopt the first live batch's
   seq as baseline (sound: lanes are ordered+reliable; the walk covers everything earlier).
3. **fsx clean: 3 seeds × 1500 ops** (pwrite/truncate/read-verify/close-reopen vs an in-RAM
   model) over the mount, ~1.3 s per run — byte-exact data path end-to-end.
4. **pjdfstest run + expected-fail manifest committed** (docs/pjdfstest-manifest.md): utimensat
   10/10 PASS; remaining failures classified (single-user squash permission matrix, chown
   ignored by design, LINK/MKNOD unimplemented + their type-loop cascades). The suite caught
   **four real server bugs, all fixed**: ACCESS gating that broke owner-chmod (`chmod +w` on
   r/o files failed), >255-byte names desyncing replies instead of NAMETOOLONG, MKNOD/LINK
   bare-status replies (client EBADRPC), and setattr requiring read access (0222-file chmod).
   All verified live on the mount after the fixes.

Also delivered: docs/mounting.md (when user mount_nfs works, failure modes, sudoers-scoped
privileged fallback; mount errors now point at it).
