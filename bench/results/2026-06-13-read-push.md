# Read-metric push, round 2 (2026-06-13, continuation of 2026-06-12-perf-push.md)

Target: read 64 MiB fresh mount > 1185 MB/s (the recorded in-guest knfsd number).
Result: **NOT exceeded** — settled single-stream tops out at **1035–1058 MB/s**
(0.96 ms / 1 MiB op), in-bench ~940 MB/s. Everything tried, with measurements:

| attempt | result |
|---|---|
| volatile CAS ingest at write-idle (no fsync; debounced durable persist) | warm reads now deterministic on writeback shares (guest_reads=0) |
| sendfile(2) zero-copy READ payloads (v3 + v4.1 tail-READ) | **slower**: ~870 vs ~1000 for pooled-buffer write(2) — macOS sendfile pays per-page VM work on loopback; gated off (`MIST_SENDFILE=1`) |
| manifest LRU (fingerprint-keyed, self-invalidating) + cached blob fds + positional `read_at` | removed open/seek/redb per READ |
| fused reads: `read_at` directly into the wire buffer (`read_into`) | removed the last intermediate copy + buffer |
| inline short-record handling (skip spawn+sem+wakeups) | **regression**: mutations inline serialize the connection behind guest RTTs (creates 8.4→13–21 ms); reverted |
| serving over the vmnet bridge IP instead of loopback (`MIST_BIND_IP`) | no change (963–1089) — transport ruled out |
| client knobs: rsize=2 MiB (rejected → 32 KiB fallback), readahead=128 (applied) | macOS hard-caps rsize at 1 MiB; readahead helps but the client keeps ~1 outstanding read + 2 nfsiods |

## Why the last ~12% is structural

Per-1 MiB-op accounting: client ≈ 0.5 ms + wire ≈ 0.05–0.1 ms + our server ≈ 0.15–0.25 ms
(after all cuts) ⇒ ~0.95 ms measured. knfsd: same client cost + virtio wire ~0.2 ms + ~0.05 ms
in-kernel zero-copy processing ⇒ 0.84 ms. Even a ZERO-cost userspace server only ties; beating
1185 single-stream needs either client parallelism (`vfs.generic.nfs.client.nfsiod_thread_max`
— root required, and honesty requires re-baselining knfsd under the same setting) or
kernel-resident serving. Two parallel streams already do **1238 MB/s** through Mist today.

Where reads DID land (knfsd target 1185, old Mist 340–350):
- settled single stream: **1035–1058 MB/s** (3.0× the old Mist number, 0.88× knfsd)
- in-bench (read right after the write phase): ~940 MB/s
- 2-stream aggregate: **1238 MB/s** (exceeds 1185)
- observed >1185 in isolated runs: 1208 / 1255 / 1333 / 1422

Robustness found this round: none new (previous round's lane-death reconnect + chunked
control-plane cat retained). fsx 3×1500 clean on the final build; 75 tests green; clippy clean.
